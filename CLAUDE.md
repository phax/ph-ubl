## Project Overview

ph-ubl is a Java library providing JAXB bindings for OASIS UBL (Universal Business Language) versions 2.0 through 2.4, plus regional variants (Turkish UBL-TR, Peruvian UBLPE, Colombian DIAN, EU eForms). It is a multi-module Maven project with 16 modules.

## Build Commands

```bash
mvn clean install                          # Full build (generates JAXB code, compiles, tests, installs)
mvn test                                   # Run all tests
mvn test -pl ph-ubl21                      # Run tests for a single module
mvn test -pl ph-ubl21 -Dtest=UBL21FuncTest # Run a single test class
mvn test -pl ph-ubl21 -Dtest=UBL21FuncTest#testReadAndWriteInvoice  # Single test method
mvn process-sources                        # Generate JAXB sources only
```

Java 17 minimum. CI tests against 17, 21, and 25.

## Architecture

**JAXB Code Generation:** Each UBL version module (ph-ubl20 through ph-ubl24) contains XSD schemas in `src/main/resources/external/schemas/` and JAXB binding files in `src/main/jaxb/`. The `jaxb-maven-plugin` generates Java classes into `target/generated-sources/` during the build — these are NOT committed. The `ph-jaxb-plugin` adds equals, hashCode, clone, toString, JSpecify annotations, and serialization support to generated classes.

**Marshaller pattern:** Each version module exposes a unified marshaller class (e.g. `UBL21Marshaller`) extending `GenericJAXBMarshaller<T>` from ph-jaxb. These provide `read()`, `write()`, `validate()`, and `getAsDocument()` for all document types in that version.

**Document type enums:** Each version has an `EUBL2xDocumentType` enum listing all supported document types (Invoice, CreditNote, Order, etc.) with metadata for schema location and root element.

**Codelist modules:** Optional `-codelists` modules contain auto-generated enums for UBL code lists (currency, country, language codes, etc.), generated by utilities in `ph-ubl-api`.

**Test resources:** `ph-ubl-testfiles` contains real UBL XML documents for all versions and document types, accessed via `MockUBL2xTestDocuments` helpers.

## Module Layout

- `ph-ubl-testfiles` — shared test XML documents
- `ph-ubl-api` — base API and code generation utilities
- `ph-ubl{20,21,22,23,24}` — JAXB bindings per UBL version
- `ph-ubl{20,21,22,23,24}-codelists` — codelist enums per version
- `ph-ubltr` — Turkish e-Invoice variant
- `ph-ublpe` — Peruvian e-Invoice variant
- `ph-ubl-dian` — Colombian e-Invoice variant
- `ph-ubl-eforms` — EU eForms (UBL 2.3 based)

## Key Dependencies

All from the `com.helger` ecosystem: `ph-commons` (collections, XML, JAXB utilities), `ph-xsds` (XSD schemas), `ph-sbdh`, `ph-genericode`. Do not introduce external alternatives.

## Testing

JUnit 4. Test classes follow patterns `*FuncTest` (functional read/write/validate cycles) and `SPITest` (SPI validation). Test utility support via `ph-unittest-support-ext`.
